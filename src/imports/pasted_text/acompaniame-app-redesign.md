Actúa como un diseñador senior UX/UI especializado en aplicaciones educativas móviles para docentes de primaria en México. Quiero que rediseñes y amplíes la app “Acompáñame” como si fuera un producto real listo para desarrollo. El diseño debe ser moderno, profesional, muy funcional, visualmente atractivo, colorido pero ordenado, fácil de entender y pensado para el uso diario de maestros dentro del salón.

La app está enfocada en:
- asistencia con NFC
- registro de actividades
- registro de calificaciones
- seguimiento por grupo
- evaluación escolar basada en Campos Formativos

OBJETIVO GENERAL
Quiero una propuesta visual completa y coherente donde la app ya no use materias tradicionales, sino Campos Formativos, y donde exista una forma clara, rápida y dinámica de registrar actividades, participación y calificaciones, tanto manualmente como con modo NFC continuo.

CAMBIO 1: REEMPLAZAR MATERIAS POR CAMPOS FORMATIVOS
En toda la app reemplaza las materias como Matemáticas, Español, Historia o Ciencias por los 4 Campos Formativos oficiales:

- Lenguajes
- Saberes y Pensamiento Científico
- Ética, Naturaleza y Sociedades
- De lo Humano y lo Comunitario

Este cambio debe verse reflejado en:
- dashboard principal
- pestañas
- filtros
- formularios
- registros
- tablas
- reportes
- accesos rápidos
- resúmenes de actividades
- calificaciones

Asigna una identidad visual clara para cada Campo Formativo:
- Lenguajes: naranja cálido
- Saberes y Pensamiento Científico: azul
- Ética, Naturaleza y Sociedades: morado
- De lo Humano y lo Comunitario: verde

Cada campo formativo debe poder seleccionarse fácilmente con tabs, chips o botones superiores.

CAMBIO 2: NUEVA PANTALLA DE REGISTRO DE ACTIVIDADES Y CALIFICACIONES
Diseña una pantalla principal llamada:
“Registro de Actividades”
o
“Registro de Actividades y Calificaciones”

Esta pantalla debe estar inspirada en un formato escolar de concentrado docente, como una tabla visual colorida y organizada, parecida a un registro de tareas escolar, pero adaptada a una app móvil moderna.

La pantalla debe incluir:
- encabezado con nombre de la app “Acompáñame”
- nombre del maestro
- grupo actual
- periodo o semana
- selector del Campo Formativo
- buscador de alumno
- botón de filtro
- botón “Agregar actividad”
- selector para cambiar entre “Vista resumen” y “Vista detalle”

La vista debe ser útil para registrar múltiples actividades del grupo y consultar rápidamente el avance por alumno.

Debe haber dos vistas:

1. Vista resumen
Mostrar por alumno:
- número de lista
- nombre del alumno
- total de actividades registradas
- total de actividades calificadas
- total de pendientes
- promedio si aplica
- indicador visual de seguimiento

2. Vista detalle
Mostrar por alumno:
- número de lista
- nombre del alumno
- nombre de la actividad
- campo formativo
- tipo de registro
- estado
- fecha
- calificación si existe
- observación breve o evidencia

El diseño debe sentirse como una herramienta real para trabajo docente diario.

CAMBIO 3: MODOS DE REGISTRO DINÁMICOS
Diseña una experiencia clara para crear una nueva actividad y registrar a los alumnos de diferentes maneras.

Crea una pantalla o modal llamado:
“Nueva actividad”

Debe incluir los siguientes campos:
- Campo formativo
- Nombre de la actividad
- Fecha
- Tipo de registro
- Método de captura
- Escala de evaluación si aplica
- Observación general

TIPOS DE REGISTRO
El maestro debe poder elegir entre:
- Solo registro
- Participación
- Calificada

MÉTODOS DE CAPTURA
El maestro debe poder elegir entre:
- Manual
- NFC continuo

COMPORTAMIENTO DINÁMICO DE LA INTERFAZ

A. Si el maestro selecciona “Solo registro”
La interfaz no debe mostrar campo para calificación numérica.
Debe permitir registrar estados como:
- Entregado
- Participó
- Cumplió
- Pendiente
- No cumplió

B. Si el maestro selecciona “Participación”
La interfaz debe permitir marcar estados rápidos como:
- Participó
- Participó con apoyo
- No participó

C. Si el maestro selecciona “Calificada”
La interfaz debe mostrar una sección adicional para configurar escala de evaluación.
Permitir elegir entre:
- 5 a 10
- 0 a 10
- Logrado / En proceso / Requiere apoyo

Después, al registrar alumnos, debe verse una tabla rápida donde el maestro capture la calificación individual por alumno.

D. Si el maestro selecciona método “NFC continuo”
Diseña una pantalla especial llamada:
“Actividad activa”
o
“Registro continuo con NFC”

Esta pantalla debe mostrar:
- nombre de la actividad abierta
- campo formativo
- grupo
- fecha
- estado del lector: NFC activo
- contador de alumnos registrados, por ejemplo 18/30
- lista de últimos alumnos escaneados
- confirmación verde cuando un alumno se registra correctamente
- advertencia amarilla o roja si el alumno ya estaba registrado o no pertenece al grupo
- botón grande “Finalizar actividad”
- botón secundario “Cambiar a captura manual”

La experiencia debe sentirse rápida, moderna y útil para el trabajo en aula. El lector NFC debe estar pensado como captura continua, donde los niños van pasando uno tras otro a tocar el celular y el sistema registra automáticamente su actividad sin tener que abrir alumno por alumno.

E. Si el maestro selecciona método “Manual”
Diseña una pantalla donde aparezca la lista del grupo en forma de tabla para capturar uno por uno:
- número de lista
- nombre del alumno
- estado o calificación
- observación individual opcional

Si la actividad es calificada:
- mostrar campo o selector de nota por alumno

Si la actividad es solo registro o participación:
- mostrar botones visuales o chips de colores para marcar estado rápidamente

ESTILO VISUAL GENERAL
La app debe verse:
- educativa
- colorida
- profesional
- moderna
- clara
- dinámica
- muy fácil de usar para docentes

Usa:
- tarjetas con esquinas redondeadas

- iconografía clara

- componentes reutilizables

COLORES SEMÁNTICOS
- Verde: entregado, registrado, correcto
- Amarillo: pendiente, en proceso, advertencia
- Rojo: no entregado, error, no registrado
- Azul: acción, navegación, información

IMPORTANTE
La propuesta debe sentirse como una app real para primaria en México.
No quiero una interfaz corporativa ni genérica.
Quiero una app alegre, escolar, funcional y enfocada en simplificar el trabajo del maestro.
El diseño debe considerar grupos de aproximadamente 30 alumnos.
Debe verse bien en móvil y también sugerir adaptación para tablet.
Debe priorizar rapidez de captura, claridad visual y facilidad para consultar información.

REFERENCIA FUNCIONAL
La lógica de uso debe permitir:
- crear una actividad
- elegir el campo formativo
- elegir si será solo registro, participación o calificada
- elegir si la captura será manual o con NFC continuo
- registrar fácilmente a todo el grupo
- consultar después el resumen por alumno y por actividad

Genera mockups realistas, consistentes entre sí, con diseño coherente de app educativa lista para evolucionar a desarrollo.
